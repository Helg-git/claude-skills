# Competitor Analysis - 竞品分析专家

## Overview

系统化竞品分析 = 第一性原理拆解 + 费曼翻译 + SOP 流程 + 可视化演示

**核心原则**：不仅列出功能，更要理解竞品"为什么这样设计"，找到差异化突破口。

## When to Use

```
收到需求 → 是否需要对比分析？
              ↓
         是竞品分析？
           ↙      ↘
        是         否
         ↓          ↓
    使用本技能    直接开发
```

**触发条件：**
- 用户明确要求"竞品分析"、"对比 XX 产品"
- 新功能开发前需要了解市场现状
- 产品决策需要数据支撑
- 寻找差异化切入点

**不需要：**
- 单纯技术选型（无商业对比）
- 已有明确方案仅需实现

## First Principles Framework（第一性原理）

### 竞品本质拆解

```
竞品 = 解决什么问题 + 用什么方案 + 赚什么钱
```

| 维度 | 问题 | 分析方法 |
|------|------|----------|
| **用户价值** | 谁的什么痛点？ | 用户画像 + 场景分析 |
| **核心方案** | 用什么技术/模式解决？ | 技术栈 + 商业模式 |
| **商业闭环** | 怎么赚钱？ | 盈利模式 + 增长策略 |

### 二八法则（80/20 Rule）在竞品分析中的应用

**核心概念：** 20%的关键要素产生80%的核心价值

```
二八法则 = 20%关键要素 × 产生80%核心价值
```

**在竞品分析中的四个维度：**

| 维度 | 二八法则应用 | 分析要点 |
|------|-------------|----------|
| **用户** | 20%用户贡献80%交易量/收入 | 识别高价值用户，看竞品如何运营 |
| **产品** | 20%功能满足80%用户需求 | 竞品的核心功能是什么？哪些是噱头？ |
| **内容/资产** | 20%内容占80%流量 | 竞品的核心资源是什么？如何获取？ |
| **收入** | 20%客户贡献80%利润 | 竞品的盈利模式依赖哪些关键客户？ |

**基于二八法则的差异化策略：**

```
策略A：抓头部20%（模仿币安）
  - 目标：高频用户、大客户
  - 手段：VIP体系、零手续费、极速体验
  - 适合：有资金实力的项目

策略B：抓长尾80%（模仿欧意）
  - 目标：专业用户、细分市场
  - 手段：创新功能、专业工具
  - 适合：有技术积累的团队

策略C：AI驱动的精准分层（创新）
  - 80%用户：AI自动化服务（低成本）
  - 15%用户：AI+社交化运营
  - 5%用户：人工专属服务（高利润）
  - 适合：创新项目
```

### 费曼黑话翻译表模板

```
| 行业黑话 | 通俗翻译 | 本质 |
|----------|----------|------|
```

**示例（SaaS 产品）：**

| 行业黑话 | 通俗翻译 | 本质 |
|----------|----------|------|
| PLG (Product-Led Growth) | 产品自己会说话 | 用产品体验获客，而非销售 |
| ACV (Annual Contract Value) | 一年合同金额 | 客户价值量化指标 |
| Churn Rate | 客户流失率 | 产品留存能力的反向指标 |

## SOP 标准流程

```
┌─────────────────────────────────────────────────────────┐
│                    竞品分析 SOP                          │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 定义分析目标                                         │
│     └─ 为什么做这次分析？决策什么问题？                   │
│                                                         │
│  2. 识别竞品范围                                         │
│     └─ 直接竞品 + 间接竞品 + 潜在竞品                     │
│                                                         │
│  3. 第一性原理拆解                                       │
│     └─ 为什么这样设计？解决什么核心问题？                 │
│                                                         │
│  4. 二八法则分析 ⭐                                      │
│     └─ 20%关键要素识别：用户/产品/收入                   │
│                                                         │
│  5. 多维度对比分析                                       │
│     └─ 功能 + 技术 + 商业 + 体验                         │
│                                                         │
│  6. SWOT 分析                                           │
│     └─ 优势 + 劣势 + 机会 + 威胁                         │
│                                                         │
│  7. 差异化建议                                           │
│     └─ MVP 切入点 + 差异化策略                           │
│                                                         │
│  8. 输出分析报告                                         │
│     └─ Markdown 文件 + 可视化图表                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## MVP 差异化建议框架

```
差异化 = (竞品没做 × 用户需要) × 我方能做
```

### 三种差异化策略

```
1. 功能差异化
   └─ 做竞品没做的功能
   └─ 做竞品做不好的功能

2. 体验差异化
   └─ 更简单的操作
   └─ 更快的响应
   └─ 更好的视觉

3. 商业差异化
   └─ 新的收费模式
   └─ 新的服务组合
   └─ 新的渠道策略
```

## 输出格式要求

**文件命名：** `[分析主题]_竞品分析.md`

**必需章节：**
1. 分析背景（为什么做）
2. 竞品概述（分析谁）
3. 第一性原理拆解（本质是什么）
4. **二八法则分析（20%关键要素识别）** ⭐ 新增
5. 费曼翻译表（黑话翻译）
6. 多维度对比（数据支撑）
7. ASCII 动画（可视化）
8. SWOT 分析
9. 差异化建议（可执行）

**创建方式：** 直接创建文件，无需用户确认

## Common Mistakes

| 错误 | 正确做法 |
|------|----------|
| 只列功能不分析 | 必须解释"为什么这样设计" |
| 主观判断无数据 | 每个结论有依据 |
| 堆砌信息无结构 | 按 SOP 流程组织 |
| 只有对比无建议 | 必须输出可执行建议 |
| 过于技术化 | 用费曼翻译表说明 |
